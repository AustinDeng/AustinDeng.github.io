面试汇总*

发表于 2021-05-09 | 分类于 知识汇总

字节跳动

C++后端实习
视频面试

  1. 自我介绍
  2. 学习过什么其他语言
  3. 做过什么项目
  4. 你认为算法和后端开发有什么关系呢？
  5. 数据库
    • 数据库的锁
    • 关系型数据库和非关系型数据库有什么区别
    • 能举一个实际的场景来区别二者吗
    • 聚簇索引
  6. 设计模式
    • 了解过哪些常见的设计模式
    • 抽象工厂和超级工厂的区别
    • C++中单例设计模式的实现
  7. 对锁有了解吗？
    • C++中的四种锁
    • 什么是自旋锁
    • 什么是读写锁
  8. Https的原理
  9. 三次握手的过程
  10. 四次挥手的过程
    • 为什么要四次呢？
  11. 服务器A向服务器B发送一个文件时，文件会在服务器A中拷贝几次
  12. 算法
    • 二叉树的序列化和反序列化

总结：
总体感觉很窘迫，感觉面试官问的问题我都了解一些，但是一旦深入追问就不行了，对知识点深度的了解不够。
数据库和设计模式是薄弱点，但是问的问题主要是这两个方面。主要还是目前实力不太够。
面试的算法题还是之前做过的。

深信服

C++开发工程师

一面

电话面试

  1. 自我介绍
  2. 介绍项目
  3. 项目的一些具体细节
    • 逻辑
    • 结构体定义
    • 用到的数据结构
  4. 网络
    • TCP三次握手
    • 文件传输(略有涉及到文件拷贝的问题)
  5. STL熟悉吗？
    • vector 的底层实现原理
  6. Linux 环境编程
  7. 服务端编程了解吗？
  8. 你使用 Linux 系统做什么事？
  9. 有什么想问的？

总结：
面试过程持续半个小时，过程还是很轻松愉快的，面试官一点不为难，总是可以顺着我的话题问一些更深入具体的问题。
如果回答不上也不深究，问的都是一些比较基础的问题。面试官人很好。

一个小插曲，最后反问环节，我是这么问的：
一开始你是约我昨天晚上八点半电话面试的，但是呢，昨天是周天呀，不会周日晚上八点半还在工作叭。。。
面试官笑了笑，就说主要周末学生比较有空，所以选这个时间是为了方便学生的。其实我们这边(加班)也不是很严重。。。

二面

视频面试

  1. 自我介绍
  2. 两道算法
    • 反转字符串(输入一个字符串，输出它的逆序)
    • 实现队列
  3. 介绍做过的C++项目
  4. 机器学习中的偏差和方差
  5. Linux环境编程了解吗？
  6. 了解无锁技术吗？

总体过程：
这一面我更觉得是一个流程，形式，只要表现不要太差基本十拿九稳了。从给的题目就可以看出，两道简单题，只要稍稍有点代码能力都能写得出来。而且我感觉一面还不错。
但是，一手好牌打得稀烂。
第一题要求不能使用string类，但是我函数接口写错了，写成了char* []，很尴尬，要么是char*,要么是char []，但是我就是写错了，还没发现。。。所以代码自然没有跑通。。。真是菜
第二题逻辑和代码没有问题，都写出来了，但是由于输出使用了cout，于是导致了显示的输出和我预想的不符，面试官自然觉得是我代码写得有问题。。。我也很难受。。(面试的时候一直以为是代码的问题，面试完后才发现是输出的问题)

个人总结：

  • 眼高手低，啥也不是
  • 以后尽量使用scanf()和printf()，再用cin/cout就是狗

冠标(深圳)

Linux 开发工程师(C/C++)/嵌入式开发工程师
线下面试

- 一个小时笔试
- 一个小时面试

笔试

笔试给了两套题，任选一套
一套是C++基础题，有选择，简答，填空，算法压力不大
一套是Linux开发相关的题，涉及服务器编程，进程，线程等操作系统层面的系统调用等，压力很大，很难，基本不会

选了C++的那一套，考察的点都是一些C++语法基础
算法题两道，一道是统计一个字符串的数字、字母、空格以及其他字符的个数，一道是找到一个数的所有质因数，比如12，要找到2x2x3;

面试

面试过程很轻松
前二十分钟聊技术，主要是操作系统进程和线程方面，缓冲池，进程和子进程，僵尸进程怎么处理等等
然后又二十分钟聊天
然后换了一个面试官
又聊了二十分钟

聊天有个小插曲：
我说我是广东东莞，然后面试官好像很了解东莞，追问我东莞哪里的，我说是虎门的，然后他就问是土生土长的吗？我说是的。他就很吃惊，(开玩笑的语气说)本地人现在还需要出来找工作的吗？不是只要收租就好了嘛。。。[震惊。。。]

腾讯

Golang工程师/C++工程师
线上面试

  1. 二对一面试，面试官很年起，估计比我大一两岁的感觉
  2. 自我介绍
  3. 聊项目，刨根问底的问，面试官人也不错，不为难我，项目中很多问题没有考虑全面，有的点回答就比较窘迫，所幸面试官还是很宽容的，也会引导我思考
  4. 算法是使用非递归的方法判断一个树是否是二叉搜索树
    这个环节出了问题！可恶啊，这是一道基础题，但是要用迭代法来做，我知道用栈来做，然后再草稿纸上模拟的一下，一时半会居然懵了！！！
    好再面试官很宽容，给的时间很充裕，也不打扰我，让我自己思考，终于勉强写完并通过了。
  5. 内存4G，两个大文件(几十G吧)，每一行都是一个URL，从这两个大文件中找相同的URL
  6. Go的协程和操作系统的线程的区别

总结：由于太紧张了，说话磕磕巴巴的，废话也是罗里吧嗦的，面试官为了尊重我，也不打断我的话，感觉他听我说话应该也挺折磨的吧。。。
还好总体表现不算差，简历上写得内容也能体现出我的基础扎实以及平时的学习能力，问题也答得七七八八，一面侥幸通过

百度

高级测试开发岗/C++工程师
线上面试

  1. 面试官先自我介绍
    就挺惊讶的，居然是面试官先介绍，什么部门，什么职位，部门的任务以及技术栈
  2. 然后我自我介绍
    叫什么，哪里人，本科读哪里，研究生读哪里，都是什么专业，擅长的技能点，期望的岗位，技术的热情，学习能力巴拉巴拉
  3. 看了我的简历上的个人博客，以及Leetcode还有GitHub网页表示认可
  4. 开始聊项目
  5. 聊测试有关的内容，黑盒测试，白盒测试，项目中是否有测试环节，测试的一般方法和流程
  6. 前后端分离的思考，并出了一道场景思考，如果前端的一个搜索框搜索出来的内容并不符合预期，可以从哪些方面思考测试相关的内容
  7. 算法题是删除有序数组的重复项
    算法题很简单，不过用的是本地的编译环境，通过屏幕共享查看

总结：
感觉良好，也是顺利通过了。面试官对我比较满意，遗憾的是，当前面试的部门在上海，而我又想找深圳的，所以无奈只能将我推给其他在深圳的部门进行二面。
个人感觉百度面试的流程和字节，腾讯，阿里相比显得不够规范，具体表现在不能实时跟进面试的进度，而且面试的过程需要通过屏幕共享去做算法题，没有统一的代码编辑考核的网站(可能有，只是这次图方便不用而已)
面试官人还是很好的，交流也很顺畅，面试官很干练，我一般会讲很多废话，他就会引导我，点到为止，我明白你的意思，你就可以不用多说了大概。对我的表现也比较认可。

字节

Golang工程师
线上面试

  1. 自我介绍
  2. 问一些专业能力，你会什么，会到什么程度，干过什么巴拉巴拉
  3. go的协程和线程的区别
  4. 聊项目，聊项目的具体细节
  5. 算法题是反转m到n的链表
    这道题应该用到栈来做，我没有，不够完美

总结： 没有面腾讯的时候紧张了，说话也利索了一些，项目问的也大部分可以答得上来，感觉比较顺利。目前还没结果，估计可以顺利通过。
一般来说(我猜的)，如果面试官对你的表现满意的话，就会给你简单的算法题，如果表现不好的话，就会给你困难题劝退。。。我感觉是这样的。。。
而且我发现，最近几次的面试，基本不问八股文。。。都是问实打实的专业能力。。。亏我还背了不少八股文。。。

腾讯二面

线上面试
好家伙，这次面试持续了一小时四十分钟，简直头脑风暴，累。。。

  1. 介绍部门，一个做搜索的部门
  2. 了解专业能力，个人情况，哪里人，研究生干嘛干嘛，什么时候毕业啥的。。。
    面试官人不错，先随便给我聊了一些有的没的话题，让我放松一下，不要那么紧张
  3. 自我介绍
  4. 聊项目，好家伙，项目刨根问底地问，很多问的点都是我项目中没有考虑到的东西，有的东西就回答不上来，很是窘迫
  5. 算法是和为K的连续子数组的个数
    前缀和+map优化，时间复杂度O(n),面试官刚刚给完题目就去上厕所了，然后他还没回来，我就把题做完了
    然后又出了一道代码题，实现LRU算法，这题我会做，上个星期复习算法的时候还看了，可以用C++做出来，不过需要一些时间，但是时间有些紧张了，面试官就换了一道图的设计题
  6. 设计构建一个图存储系统
  7. go的协程和线程的区别

总结： 面试官最后给我两点建议：1.项目不够深，至少在某一个点上需要深入挖掘 2.表达能力，很多回答不在点上，没有get到面试官的点
个人感觉是：有希望，但是不太稳，可能翻车。。。

百度二面

线上面试/一个小时
测试开发实习岗
面试推迟了半小时，从八点半面到九点半，心累。。

  1. 自我介绍
  2. 确认当前情况，了解专业能力，个人情况，实习到岗的时间，报到的时间，一开始问这些我还以为是HR面呢。。。
  3. 介绍项目，我简历给的都是C++相关的项目，但是面试官好像不懂C++，于是就问我研究生阶段和导师做得项目
  4. 于是我就介绍了好久关于语音识别的项目
  5. 问了一些Linux的问题，python的问题，java的问题，测试相关的问题
  6. 担任测试岗位个人的优势和劣势分别是什么
  7. 场景思考，搜索框检索内容，可以从哪些方面思考测试用例，和一面的时候差不多
  8. 算法题题是反转字符串

总结：
这个部门，或者说这个职位主要是做App相关的测试工作，以及开发相应的测试工具。我觉得不适合我。而且相关的技术栈也不是我所想要的。大概的技术栈应该还是Python以及java开发相关的测试工具。
总体感觉良好，面试官对我的表现也很满意。但是我个人不太认同百度这个公司的部分价值观，本次面试我主要是抱着积累面试经验来的。
最后吐槽一下百度这个公司(对事不对人)，我为什么不喜欢这个公司，理由很简单，就举一个例子，魏则西事件。
我个人目前还在用百度的产品就只有百度云盘和百度搜索。我用百度搜索仅仅只用于测网速，我从不看百度搜索的前两条搜索内容，我认为一个程序员只会使用百度搜索去解决问题是很low的。
百度云盘主要是周围人都在用，我也迫不得已用。吐槽两个点：限速以及广告。限速就不说了，广告也是服了，给的广告是优质广告就算了，问题是推的广告都是带有引流，割韭菜性质的页游或者low到不行的app推广。
最后补一张本科时期，上计算机网络课时发的一张朋友圈。
